  • Java

    We have a Java library that simplifies accessing the Entity dataset. Download the precompiled binary (jar file) or check out the source bundle.

    Depending on your Java programming environment, for instance, Processing or an IDE like Eclipse or IntelliJ, you need to import the library first before you can use it. In Processing, you need to download the jar file linked above and then put it into a "code" folder next to the Processing pde file that you want to use the library with. After that restart Processing (for safe measure) and use the library in your code. 

    				  
// import the library
import nl.tue.id.datafoundry.*;
// fill in the Data Foundry server URL, just the domain (add port if needed)
DataFoundry df = new DataFoundry("server.url.com");
// create dataset access
DFDataset iot = df.dataset(@dataset.getId(), "@dataset.configuration(Dataset.API_TOKEN, "TOKEN")");
// log activity to IoT dataset
iot.device("DEVICE_ID").activity("ACTIVITY").log();
// example with DEVICE_ID = "d1234567890" and ACTIVITY = "indoor_climate"
iot.device("d1234567890").activity("indoor_climate").log();
// same example, but with additional data ("temperature" and "door")
iot.device("d1234567890").activity("indoor_climate").data("temperature", 34).data("door", "open").log();
// log new data, reuse the dataset access with device and activity set already:
iot.data("temperature", 32).data("door", "closed").log();

    Replace DEVICE_ID by the refId attribute of any device in the project. Use any ACTIVITY or leave empty. Add data as needed. Don't forget to send everything to Data Foundry with log() at the end. You can then reuse the DFDataset object (here: iot) to send more events or data items.
